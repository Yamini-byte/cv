Optimization of the code is often performed at the end of the development stage since it reduces readability and adds code that is used to increase the performance.

Answers to the above questions can be given as

  1. the outcome of the intermediate code generator is mostly the three address code. The other way the optimized code can be represented is are broadly
    1. Linear form
      • postfix expressions
      • three address code
    2. tree form
      • syntax tree
      • directed acyclic Graph

  2. The code optimization is the part of syntheis phase of the comiler,which is a program transformation technique, which tries to mprove the intermediate code making it consume fewer resources(i.r., CPU, Memory) so that faster running machine code will result.

  3. The code optimization process should meet the following objectives-
    • The optimization must be correct, it must not in any way change the meaning of the program.
    • Optimization should increase the speed and performance of the program
    • The compilation time must be kept reasonable
    • the optimization must no delay the overall compilation process.

  4. The types of code optimizations can be broadly classified into two types-
    • Machine Independed Optimization-Does not involve any CPU registers ot absolute memory Locations.
    • Machine Dependent Code optimization-done after the target code has been generated and when code is transformed according to the target machine architecture.

    2. x = op z

    4. Conditional goto- if g goto L
    5. Unconditional goto- goto L
    6. A[i] = x and y = A[i]
    7. x = *p and y = &p

    1. Quadruples
    2. Triples
    3. Indirect triples
